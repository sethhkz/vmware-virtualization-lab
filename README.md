# VMware Virtualization Lab - Multi-OS Testing Environment

## Overview
This project involved creating a virtualization lab using VMware Workstation to run and manage multiple operating systems from one physical computer. The goal was to build a safe testing environment where different systems could be installed, configured and tested without affecting the main host machine.

## Objective
Build a working virtual lab where multiple operating systems could be installed and managed safely. Focus on understanding how virtual machines are created, how CPU, RAM and storage are allocated, and how network settings affect connectivity.

## What I Built
- Windows virtual machine (guest OS for testing Windows configuration)
- Kali Linux virtual machine (Linux-based environment for command-line practice)
- Virtual hardware configuration (CPU allocation, RAM, storage)
- NAT and bridged networking setup
- Isolated system environments for technical experimentation

## Key Skills Demonstrated
- Virtual machine creation and configuration
- CPU and memory resource allocation
- Virtual disk storage management
- NAT networking (VMs accessing internet through host)
- Bridged networking (VMs behaving as normal LAN devices)
- Operating system installation and setup
- Virtual hardware troubleshooting
- Cross-platform system management (Windows & Linux)

## Learning Outcomes
- How to safely test system configurations without risking the main computer
- Understanding virtual hardware vs physical hardware
- Network behavior differences between NAT and bridged modes
- Managing multiple isolated operating systems
- Practical experience with both Windows and Linux environments

## Tools Used
- VMware Workstation
- Windows (guest OS)
- Kali Linux (guest OS)
- Virtual CPU allocation
- Virtual memory allocation
- Virtual disk storage
